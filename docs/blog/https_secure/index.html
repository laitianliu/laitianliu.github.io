HTTPS 机制原理

2023-03-092 分钟阅读 银河之畔 技术 网络Https

网络传输安全性

  • http 协议：不安全未加密

HTTPS 协议是由 HTTP 加上 TLS/SSL 协议构建的可进行加密传输、身份认证的网络协议，主要通过数字证书、加密算法、非对称密钥等技术完成互联网数据传输加密，实现互联网传输安全保护。

  • https 协议： 安全，对请求报文和响应报文做加密

对称加密与非对称加密

对称加密

  • 特点
    加解密使用相同密钥
    高效，适用于大量数据的加密场景
    算法公开，安全性取决于密钥大小，但是密钥越大效率越低，需要在安全和效率中权衡

  • 缺点
    算法本身安全，但是使用场景不够安全，因为解密和加密都是同一个密钥

非对称加密

  • 使用 匹配的一对密钥 来 分别 进行加密和解密，这两个密钥是公开密钥（public key, 简称公钥）和（private key, 简称私钥）
  • 注意： 公钥加密的数据 只能用 对应的私钥解密， 同理。 私钥加密的数据 只能用 对应的公钥解密
  • 算法：RSA 、ECC、Elgamal、背包算法、Rabin、D-H 等
  • 特点：安全性高
  • 缺点： 加密复杂，效率低，耗时比较长
  • 用法概要：
    加密：对数据做加密
    签名：证明数据是谁发的

非对称加密方法

  • 准备工作：
    1. 生成一对匹配的私钥和公钥（ps: 公钥其实是根据私钥生成的）
    2. 将公钥公布给外面

公钥加密

  • 场景 用来针对互联网上加密数据传递
    小红 用 小明的公钥 对数据进行加密，然后发给小明， 小明用自己的私钥解密

因为一个公钥加密的数据 只有 对应的私钥才能解密，所以密文很安全
补充：如果要在网络上相互发送密文，可以让对方将对方的公钥发过来，用对方的公钥来加密数据

私钥签名

  • 场景：目的是为了将明文公布给别人，同时证明是自己发的；作用是防止明文被篡改。
    第一步: 小明用 自己的私钥 对对明文摘要的hash值进行加密，把密文（签名）和明文一起发给小红

第二步: 小红 用Jame的公钥 进行解密， 解密后的明文hash值 和 接受到的明文hash进行对比，如果一样的则是小明发的
